# VibeLoom UI

A small, token-driven CSS design system — **crimson + cool greys, light & dark, Constructivist-influenced but readability-first.** Drop it onto any page, long-form doc, or slide deck. Everything is namespaced (`--vl-*`, `.vl-*`) so it coexists with existing site CSS.

> Crimson `#e84057` and the cool-grey neutrals are taken from **vibeloom.ai**, so this drops onto the site cleanly.

## Files

| File | What it is |
|---|---|
| `tokens.css` | Design tokens — color, type, spacing, shadow, radius — for light and `[data-theme="dark"]`. Imports the fonts. **Load first.** |
| `base.css` | Safe reset, `.vl-page` (theme surface + type for `<body>`), and `.vl-prose` content typography. |
| `components.css` | Buttons, callouts, tables, cards, pills, code, figures + diagram primitives, sources, kbd, tags. |
| `layout.css` + `layout.js` | Container, doc+sidebar layout, hero, band/part-divider, grids, utilities; + progress / mobile-drawer / scroll-spy / theme JS. |
| `slides.css` + `slides.js` | Dependency-free presentation system. |
| `theme.css` | The **bundle** (tokens+base+components+layout+slides). Generated by `build.py`. |
| `styleguide.html` | Living reference — **open this first.** |
| `slides-demo.html` | Example deck. |
| `build.py` | Rebuild `theme.css` after editing any module. |

## Quick start

One file, drop-in:

```html
<link rel="stylesheet" href="theme.css">
<body class="vl-page">
  <main class="vl-container vl-prose"> … </main>
</body>
```

Or cherry-pick modules (always include `tokens.css` first):

```html
<link rel="stylesheet" href="tokens.css">
<link rel="stylesheet" href="base.css">
<link rel="stylesheet" href="components.css">
<!-- add layout.css / slides.css only where needed -->
```

Fonts (Oswald / Inter / JetBrains Mono) load automatically via an `@import` at the top of `tokens.css`. To go fully offline, swap that `@import` for local `@font-face` rules.

## Dark mode

Set `data-theme="dark"` on `<html>`. `layout.js` and `slides.js` define `window.vlToggleTheme()` and auto-init from `localStorage`, defaulting to light (dark only when toggled). Minimal inline init if you don't use those:

```html
<script>
  (function(){var t=localStorage.getItem('vl-theme')==='dark'?'dark':'light';document.documentElement.dataset.theme=t;})();
</script>
<button onclick="vlToggleTheme()">Toggle theme</button>
```

Diagrams (`.vl-figure`) and code blocks render on fixed light/dark "islands" in both themes for legibility.

## Diagrams

Inline SVGs use short class names **scoped inside `.vl-figure`** (collision-safe): `.fill-1`…`.fill-6`/`.fill-s`, `.stroke-*`, `.solid-*`, `.txt-*`, `.edge`, `.edge-strong`, `.edge-d`, and `.dg-t` / `.dg-ink` / `.dg-center` text helpers. The arrow markers must exist once per page — copy this near the top of `<body>`:

```html
<svg width="0" height="0" aria-hidden="true"><defs>
  <marker id="vl-arrow"  viewBox="0 0 10 10" refX="8.5" refY="5" markerWidth="7" markerHeight="7" orient="auto-start-reverse"><path d="M0 0 L10 5 L0 10 z" fill="#5a5a5a"/></marker>
  <marker id="vl-arrowB" viewBox="0 0 10 10" refX="8.5" refY="5" markerWidth="7" markerHeight="7" orient="auto-start-reverse"><path d="M0 0 L10 5 L0 10 z" fill="#e84057"/></marker>
  <marker id="vl-arrowG" viewBox="0 0 10 10" refX="8.5" refY="5" markerWidth="7" markerHeight="7" orient="auto-start-reverse"><path d="M0 0 L10 5 L0 10 z" fill="#1a1a1a"/></marker>
  <marker id="vl-dot" viewBox="0 0 6 6" refX="3" refY="3" markerWidth="6" markerHeight="6"><circle cx="3" cy="3" r="3" fill="#5a5a5a"/></marker>
</defs></svg>
```

## Slides

```html
<body class="vl-page">
  <div class="vl-deck">
    <section class="vl-slide vl-slide--title">…</section>
    <section class="vl-slide vl-slide--two">…</section>
  </div>
  <div class="vl-deck__progress"></div>
  <script src="slides.js"></script>
</body>
```

Arrow keys / space navigate, **F** fullscreen, **D** theme; **print → PDF** gives one slide per page. Slide variants: `--title --section --two --quote --image --center --dark --end`. See `slides-demo.html`.

## Component cheat-sheet

- **Buttons:** `.vl-btn` + `--solid` / `--ghost` / `--ink` / `--sm` / `--lg`
- **Labels:** `.vl-kicker`, `.vl-eyebrow`
- **Callouts:** `.vl-callout` + `--take` / `--pitfall` / `--key` / `--def` / `--note` (with `.vl-callout__head` + `.vl-callout__icon`)
- **Tables:** `.vl-table-wrap` > `table.vl-table` (or any `table` inside `.vl-prose`); cell marks `.vl-yes` / `.vl-no` / `.vl-maybe`, `.vl-num`
- **Cards:** `.vl-card` (+ `--soft`), `.vl-card__title`, `.vl-card__body`; grids `.vl-cols`, `.vl-cols-3`, `.vl-grid`
- **Pills:** `.vl-pill` + `--1`…`--6` / `--s` / `--brand` / `--ink`
- **Code:** `.vl-code` (block) / `.vl-code-inline`; syntax spans `.vl-kw .vl-st .vl-fn .vl-nm .vl-nu .vl-cm`
- **Figure:** `.vl-figure` > `.vl-figure__head` (`__title`, `__num`) > `.vl-figure__body` > `figcaption`
- **Layout:** `.vl-doc` + `.vl-sidebar` + `.vl-main`, `.vl-hero`, `.vl-band`, `.vl-section`, `.vl-container`
- **Sources / misc:** `.vl-sources`, `sup.vl-cite`, `.vl-kbd`, `.vl-tags`

## Using it on vibeloom.ai

It's namespaced, so it won't collide with the site's existing `--c-*` / classes. Two paths: (1) include `theme.css` and use `.vl-` classes on new sections; or (2) unify tokens by mapping the site's vars onto these, e.g. `:root{ --vl-brand:var(--c-red); --vl-text:var(--c-ink); … }`.

## Rebranding

All tokens/classes are prefixed. To rename: find-and-replace `--vl-` → `--xx-` and `vl-` → `xx-` across the modules, then `python3 build.py`.

## Rebuild

```bash
python3 build.py    # regenerates theme.css from the modules
```
